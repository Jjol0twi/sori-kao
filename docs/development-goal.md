# 개발 기획서: 한국어 음운 특징 기반 소리 인상 해석기

## 1. 프로젝트 개요

이 프로젝트는 의성어·의태어·감탄사·구호 같은 **한국어 짧은 표현**을 입력받아 초성·중성·종성·반복·문장부호 같은 음운 특징을 분석하고, 입력이 주는 **소리 인상**을 해석하는 규칙 기반 Python 프로그램이다.

핵심 목표는 카모지 추천이 아니다. 프로젝트의 중심은 **표현의 음운 특징이 어떤 인상으로 해석되는지 설명하는 것**이며, 카모지는 필요할 때 해석 결과에 붙이는 선택적 부가 출력으로만 둔다.

**적용 범위.** 음운상징의 근거는 의성어·의태어·감탄사·짧은 표현에서 더 강하게 나타난다. 따라서 일반 문장 전체의 의미를 해석하는 도구가 아니라, 음운 인상이 분명히 드러나는 짧은 표현을 주 대상으로 한다. 길고 일반적인 문장도 입력은 받지만, 음운상징성이 약하므로 낮은 신뢰도로 보수적으로 해석한다.

## 2. 배경: 감정 단어가 아니라 소리 인상

기존의 이모티콘·카모지 추천은 명시적인 감정·상황 단어에 의존하기 쉽다. 그러나 실제 한국어 표현에는 감정 단어가 없어도 소리와 형태에서 인상이 드러나는 경우가 많다.

- `아싸아싸!!` → 반복·밝은 모음·느낌표로 밝고 리듬감 있는 인상
- `우우...` → 어두운 모음·반복·말줄임표로 무겁고 지속되는 인상
- `빠샤!` → 거센소리·느낌표로 강하고 격렬한 인상

따라서 본 프로젝트는 입력의 의미를 직접 맞히는 **감정 판정기가 아니라**, 입력 표면에 드러나는 음운 특징을 계산해 "어떤 소리 느낌이 강한가"를 설명하는 **해석기**를 목표로 한다.

## 3. 문헌 근거와 카테고리 설계 원칙

한국어 음운상징·의성어·의태어 연구는 다음을 다룬다.

- **국립국어원(1993)**: 의성·의태어의 반복 가능성, 음절 반복, 음소 교체, 모음조화에 의한 어휘 대립(깡총/껑충).
- **박동근(1999)**: 의성어·의태어의 음운 교체 말맛 — ① 음성모음이 양성모음보다 [강조](어두움·큼), ② 된소리·거센소리가 [강조]·[거침](강함·격렬함), ③ **종성 대립**: ㄱ류 폐쇄 종성 = "울림 없이 순간적으로 끝나는" 단절, ㅇ 종성 = "소리가 울려 여운"·부드러움.
- **강석한(2018)**: 음상 강화형 반복 부사어에서 종성이 주로 공명음 ㅇ·ㅁ·ㄹ로 끝나며(운율·모라 효과), 음상 강화는 모음 변화 ≫ 자음 변화 순으로 일어남.
- **성제현(2018)**: 밝은 모음과 어두운 모음의 대비가 작음·가벼움 ↔ 큼·무거움 같은 인상과 연결될 수 있음.
- **박동근(2014)**: 흉내말의 어말 자음 교체가 세부 인상 차이(변화·교체감)에 관여함.
- **Akita(2021)**: 모음-크기 상징(close/front=작음, open/back=큼)은 범언어적이나 **한국어는 예외**로, 전통 음/양(밝음/어두움) 체계를 따른다 → 밝음/어두움 축은 보편 조음 매핑이 아니라 한국어 양성/음성모음에 근거해야 함.
- **반론(Pimentel 2019; Dingemanse 2015)**: 형태-의미 체계성은 실재하나 효과크기가 작고 자의성이 지배적이며, 도상성은 언어마다 다를 수 있다 → 결정 법칙이 아니라 경향으로 다루고 신뢰도로 완화.

다만 이 문헌들은 카모지 사용 상황을 위한 감정·대화 분류표를 제공하지 않는다. 그래서 본 프로젝트는 카모지 편의에 맞춘 감정·상황 카테고리를 핵심으로 쓰지 않고, 문헌에서 가져오는 것은 감정명이 아니라 **음운 인상 축**이다.

이에 따라 카테고리 설계 원칙은 다음과 같다.

1. 감정·상황 이름이 아니라 **음운 인상 이름**을 사용한다.
2. 카테고리는 문헌에서 다루는 반복·교체·모음조화·자음 인상과 연결되어야 한다.
3. 한 입력이 여러 인상을 동시에 가질 수 있으므로 **Top-3** 후보를 출력한다.

## 4. 해석 카테고리 (이론 10축 / MVP 9축)

해석 카테고리는 **이론 범주 10축**으로 정의하되, MVP에서 실제로 점수가 산출되는 것은 **9축**이다. 자음/모음 교체 자동 탐지가 필요한 `변화·교체감`(※)은 MVP에서 점수를 내지 않는 확장 축으로 분리한다.

아래 표는 각 인상 축의 의미와, 그 축이 어떤 음운 특징(→ 구현 feature)·문헌 근거에 연결되는지를 한 곳에 정리한 것이다.

| 인상 축 | 의미 | 음운 특징 → feature | 문헌 근거 |
|---|---|---|---|
| 밝음·가벼움 | 밝고 가볍고 작게 느껴지는 인상 | 양성 모음 → bright_vowel_ratio | 박동근(1999:371), 국립국어원(1993), Akita(2021) |
| 어두움·무거움 | 어둡고 무겁고 낮게 느껴지는 인상 | 음성 모음 → dark_vowel_ratio | 박동근(1999:371), 성제현(2018) |
| 작음·섬세함 | 작고 섬세하며 약한 인상 | 양성 모음 + 약한 강자음 → bright_vowel_ratio, ↓tense/aspirated | Akita(2021), 성제현(2018) |
| 큼·둔중함 | 크고 둔중하며 무거운 인상 | 음성 모음 + 공명 종성 울림 → dark_vowel_ratio, final_ng_ratio | 박동근(1999:371), 성제현(2018) |
| 강함·격렬함 | 강하고 날카롭거나 격렬한 인상 | 된소리·거센소리 → tense/aspirated_consonant_ratio | 박동근(1999:374,376) |
| 부드러움·흐름 | 부드럽고 이어지며 완만한 인상 | 비음·유음(초성) + 공명 받침 → nasal_liquid_ratio, final_sonorant_ratio | 박동근(1999) ㅇ종성, 강석한(2018) |
| 반복·리듬 | 반복되고 리듬감 있는 인상 | 완전/부분 반복 → syllable_repetition, char_repetition | 국립국어원(1993) 중첩 도상성 |
| 지속·여운 | 길게 이어지거나 남는 인상 | 종성 울림(ㅇ·공명)·말줄임·반복 지속 → final_ng_ratio, final_sonorant_ratio, ellipsis_energy | 박동근(1999) ㅇ=여운, 국립국어원(1993) |
| 변화·교체감 ※ | 자음·모음이 바뀌며 생기는 변화감 | 자음/모음 교체 → (미구현·확장 축) | 박동근(2014), 국립국어원(1993) |
| 막힘·단절감 | 막히고 끊기는 인상 | 불파 폐쇄음 받침(ㄱㄷㅂ류) → final_stop_ratio | 박동근(1999) 종성 대립 |

※ `변화·교체감`은 교체 탐지 feature가 MVP에 없어 Top-3 산출에 포함되지 않는다. 이론 범주로만 남겨 두고, 이후 자음/모음 교체 탐지를 추가하면 활성화한다(즉 MVP 실제 산출은 9축).

**상호배타적 분류가 아니다.** 한 입력이 동시에 여러 인상을 가질 수 있다. 특히 `밝음·가벼움`과 `작음·섬세함`, `어두움·무거움`과 `큼·둔중함`은 같은 모음 계열(양성/음성)에서 함께 점수가 올라간다 — 오류가 아니라 양성·음성 모음이 밝기·크기·무게 같은 여러 인상과 동시에 연결되기 때문이다[성제현, 2018]. 그래서 결과를 단일 분류가 아니라 **Top-3 인상 묶음**으로 제시한다.

표의 **문헌 근거**는 "어떤 음운 특징을 볼지"와 "그 특징이 어느 축에 어느 방향(부호)으로 기여하는지"의 출처다. **가중치의 절대 수치**까지 논문에서 가져온 것은 아니며, 그 위상은 §7에서 구분한다.

## 5. 처리 흐름과 출력 예시

```text
표현 입력
→ 입력 전처리(한글/문장부호/표기 보조 신호 분리)
→ 한글 완성형 음절 분해(초성·중성·종성)
→ 음운 특징 추출 → 14차원 feature vector
→ 해석 카테고리별 점수 계산
→ Top-3 해석 후보 산출
→ 해석 이유와 신뢰도 출력
→ (선택) 해석 결과에 맞는 카모지 등 표시 요소 연결
```

출력 예시:

```text
입력: 아싸아싸!!

해석 후보:
1. 밝음·가벼움
2. 반복·리듬
3. 강함·격렬함

해석 이유:
- 밝은 모음 비율이 높아 밝고 가벼운 인상 점수가 올라갔습니다.
- 같은 음절 패턴이 반복되어 반복·리듬 인상이 감지되었습니다.
- 느낌표가 있어 강도 신호가 보조적으로 반영되었습니다.

신뢰도: high
```

## 6. 구현 범위

**MVP에서 구현한다.**

- tkinter 기반 입력 화면과 해석 결과 출력
- 한글 완성형 음절 분해
- 14차원 음운 특징 벡터 생성(모음·자음 분류, 종성 종류별 비율(불파 폐쇄음/ㅇ/공명음), 반복, 문장부호 강도)
- NumPy 기반 해석 카테고리 점수 계산과 Top-3 후보·신뢰도 출력
- 해석 이유 출력
- 가중치와 설정값을 JSON으로 분리

**MVP에서 제외한다.**

- 생성형 AI·LLM·외부 API·네트워크 호출
- 대규모 학습 데이터, 딥러닝 모델 학습, 사용자 개인화
- 카모지 추천을 핵심 기능으로 삼는 구조
- 자음/모음 교체 자동 탐지의 완전 구현
- 한국어 발음 변환(G2P)의 완전 구현
- 해석 결과의 심리학적 정확성 보장

## 7. 수치값과 문헌 근거의 구분

문헌은 ① **어떤 음운 특징을 볼지**와 ② **그 특징이 어느 인상 축에 어느 방향(부호)으로 기여하는지**의 근거다. `category_weights.json`의 가중치는 이 문헌적 부호·상대 크기를 따른다. 예를 들어 막힘·단절감은 불파 폐쇄음 받침(`final_stop_ratio`)에서만 양의 값을 받고 공명 받침은 오히려 여운·부드러움 쪽으로 가는데, 이는 박동근(1999)의 종성 대립("ㄱ종성=순간적 단절 vs ㅇ종성=울림·여운")을 그대로 옮긴 것이다. 마찬가지로 모음 양성/음성→밝음·작음/어두움·큼(박동근 1999:371), 된소리·거센소리→강함·격렬함(박동근 1999:374,376)도 문헌 근거를 가진다.

다만 **정확한 소수점 수치 자체**는 논문에 있는 값이 아니라 그 부호·순위 제약 안에서 사람이 정한 초기값이며, bias·신뢰도 임계값도 같은 위상의 구현 파라미터다. 또 음성상징은 결정 법칙이 아니라 효과 크기가 작은 경향이므로(Pimentel 2019; Dingemanse 2015), 점수는 절대값이 아니라 **상대 순위 + 신뢰도**로만 해석한다. 문장부호(느낌표·물음표·말줄임표) 가중치는 음운 문헌 근거가 약한 표기 휴리스틱으로 따로 둔다.

따라서 보고서·발표에서는 다음처럼 말한다.

```text
가중치의 부호와 상대 크기는 한국어 음운상징 문헌(박동근 1999의 모음·자음·
종성 말맛, 국립국어원 1993, 강석한 2018 등)에 근거한다.
다만 소수점 수치 자체는 논문 값이 아니라 그 제약 안에서 정한 구현 초기값이고,
음성상징은 경향이므로 결과는 절대 점수가 아니라 상대 순위와 신뢰도로 해석한다.
```

## 8. 평가 기준

평가는 카모지를 잘 추천했는지가 아니라, **음운 인상 해석이 설계 의도와 맞는지**를 본다.

- 해석 카테고리가 문헌 기반 음운 인상 축과 연결되어 있는가
- 출력 카테고리가 감정·상황명이 아니라 음운 인상명인가
- 해석 이유가 실제 feature vector 값과 일치하는가
- Top-3 후보로 복수 인상을 보여주는가
- 짧거나 모호한 입력을 낮은 신뢰도로 표시하는가
- 표기 보조 신호(문장부호·자모 반복)가 쓰였을 때 순수 음운 해석과 구분되는가
- 카모지가 없어도 프로그램의 목표와 결과가 설명되는가

## 9. 한계와 방어 논리

음운상징 연구는 소리와 의미 사이의 경향을 보여주지만, 그 관계가 절대적이라고 보지는 않는다. Dingemanse 등(2016)은 ideophone도 음소만으로 의미가 완전히 결정되지 않으며 운율과 관습이 함께 작동한다고 설명한다. Pimentel 등(2019)도 소리와 의미의 통계적 관련은 관찰되지만 효과 크기는 작고 자연언어에서는 자의성이 크게 작동한다고 본다.

따라서 본 프로젝트는 "음운만으로 감정을 맞히는 프로그램"이라고 주장하지 않는다. 올바른 설명은 다음과 같다.

```text
이 프로젝트는 한국어 의성어·의태어 연구에서 다루는 반복, 교체,
모음조화, 자음 인상 등을 계산 가능한 특징으로 바꾸고,
입력 표현이 어떤 소리 인상을 갖는지 설명하는 프로그램입니다.
카모지는 해석 결과를 보여주기 위한 부가 서비스입니다.
```

## 10. 확장 가능성

다음 항목은 MVP 평가 대상이 아니라 이후 발전 방향이다. 기능을 넓히더라도 핵심 목표는 카모지 추천이 아니라 음운 인상 해석이다.

| 구분 | 확장 항목 | MVP에서 제외하는 이유 |
|---|---|---|
| 확장 축 | 자음/모음 교체 자동 탐지 | `변화·교체감`을 산출하려면 별도 탐지 알고리즘이 필요 |
| 입력 분석 | 표기형과 발음형 비교 분석 | 발음 변환 규칙이 들어가면 범위가 커짐 |
| 입력 분석 | 발음 변환(G2P)을 feature 추출에만 적용 | 표기와 실제 발음 차이를 다룰 수 있으나 MVP 핵심은 아님 |
| 평가 자료 | 해석 카테고리별 예시 표현 수집 | 가중치 조정용 자료가 필요하나 과제 범위를 넘길 수 있음 |
| 출력 | 해석 결과에 맞는 카모지·기호 표현 부가 출력 | 시각적 표시일 뿐 해석 모델의 근거가 아님 |
| 출력 | 입력별 feature vector 시각화 | 발표 보조에는 좋으나 핵심 로직은 아님 |
