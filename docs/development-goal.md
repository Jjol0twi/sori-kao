# 개발 기획서: 한국어 음운 특징 기반 소리 인상 해석기

## 1. 프로젝트 개요

이 프로젝트는 의성어·의태어·감탄사·구호 같은 **한국어 짧은 표현**을 입력받아 초성·중성·종성·반복·문장부호 같은 음운 특징을 분석하고, 입력이 주는 **소리 인상**을 해석하는 규칙 기반 Python 프로그램이다.

핵심 목표는 카모지 추천이 아니다. 프로젝트의 중심은 **표현의 음운 특징이 어떤 인상으로 해석되는지 설명하는 것**이며, 카모지는 필요할 때 해석 결과에 붙이는 선택적 부가 출력으로만 둔다.

**적용 범위.** 음운상징의 근거는 의성어·의태어·감탄사·짧은 표현에서 더 강하게 나타난다. 따라서 일반 문장 전체의 의미를 해석하는 도구가 아니라, 음운 인상이 분명히 드러나는 짧은 표현을 주 대상으로 한다. 길고 일반적인 문장도 입력은 받지만, 음운상징성이 약하므로 낮은 신뢰도로 보수적으로 해석한다.

## 2. 배경: 감정 단어가 아니라 소리 인상

기존의 이모티콘·카모지 추천은 `행복`, `슬픔`, `감사`, `미안`처럼 명시적인 감정·상황 단어에 의존하기 쉽다. 그러나 실제 한국어 표현에는 감정 단어가 없어도 소리와 형태에서 인상이 드러나는 경우가 많다.

- `아싸아싸!!` → 반복·밝은 모음·느낌표로 밝고 리듬감 있는 인상
- `으으...` → 어두운 모음·반복·말줄임표로 무겁고 지속되는 인상
- `빠샤!` → 거센소리·느낌표로 강하고 격렬한 인상

따라서 본 프로젝트는 입력의 의미를 직접 맞히는 **감정 판정기가 아니라**, 입력 표면에 드러나는 음운 특징을 계산해 "어떤 소리 느낌이 강한가"를 설명하는 **해석기**를 목표로 한다.

## 3. 문헌 근거와 카테고리 설계 원칙

한국어 음운상징·의성어·의태어 연구는 다음을 다룬다.

- **국립국어원(1993)**: 의성·의태어의 반복 가능성, 음절 반복, 음소 교체, 모음조화에 의한 어휘 대립.
- **성제현(2018)**: 밝은 모음과 어두운 모음의 대비가 작음·가벼움 ↔ 큼·무거움 같은 인상과 연결될 수 있음.
- **박동근(2014)**: 흉내말의 어말 자음 교체가 세부 인상 차이에 관여함.

다만 이 문헌들은 `응원`, `감사`, `사과`, `집중` 같은 카모지 사용 상황을 감정 분류표로 제공하지 않는다. 그래서 본 프로젝트는 카모지 편의에 맞춘 감정·상황 카테고리를 핵심으로 쓰지 않고, 문헌에서 가져오는 것은 감정명이 아니라 **음운 인상 축**이다.

이에 따라 카테고리 설계 원칙은 다음과 같다.

1. 감정·상황 이름이 아니라 **음운 인상 이름**을 사용한다.
2. 카테고리는 문헌에서 다루는 반복·교체·모음조화·자음 인상과 연결되어야 한다.
3. 한 입력이 여러 인상을 동시에 가질 수 있으므로 **Top-3** 후보를 출력한다.

## 4. 해석 카테고리 (이론 10축 / MVP 9축)

해석 카테고리는 **이론 범주 10축**으로 정의하되, MVP에서 실제로 점수가 산출되는 것은 **9축**이다. 자음/모음 교체 자동 탐지가 필요한 `변화·교체감`(※)은 MVP에서 점수를 내지 않는 확장 축으로 분리한다.

| 카테고리 | 의미 | 주요 특징 |
|---|---|---|
| 밝음·가벼움 | 밝고 가볍고 작게 느껴지는 인상 | 밝은/양성 모음 |
| 어두움·무거움 | 어둡고 무겁고 낮게 느껴지는 인상 | 어두운/음성 모음 |
| 작음·섬세함 | 작고 섬세하며 약한 움직임의 인상 | 밝은 모음, 낮은 받침 밀도 |
| 큼·둔중함 | 크고 둔중하며 무거운 움직임의 인상 | 어두운 모음, 받침 밀도 |
| 강함·격렬함 | 강하고 날카롭거나 격렬한 인상 | 된소리, 거센소리 |
| 부드러움·흐름 | 부드럽고 이어지며 완만한 인상 | 비음, 유음 |
| 반복·리듬 | 반복되고 리듬감 있는 인상 | 음절 반복, 문자 반복 |
| 지속·여운 | 길게 이어지거나 남는 인상 | 말줄임표, 종성 ㅇ, 반복 지속 |
| 변화·교체감 ※ | 자음·모음이 바뀌며 생기는 변화감 | 자음 교체, 모음 교체 (MVP 미산출·확장 축) |
| 막힘·단절감 | 막히고 끊기는 인상 | 받침 밀도, 어말 자음 |

※ `변화·교체감`은 교체 탐지 feature가 MVP에 없어 Top-3 산출에 포함되지 않는다. 이론 범주로는 남겨, 이후 자음/모음 교체 탐지를 추가하면 활성화한다. 즉 **MVP 실제 산출은 9축**이다.

**카테고리는 상호배타적 분류가 아니다.** 이 축들은 하나의 정답을 고르는 분류가 아니라, 한 입력이 동시에 여러 인상을 가질 수 있는 인상 축이다. 특히 `밝음·가벼움`과 `작음·섬세함`, `어두움·무거움`과 `큼·둔중함`은 같은 모음 계열(양성/음성)에서 함께 점수가 올라간다. 이는 오류가 아니라, 음운상징에서 양성·음성 모음이 밝기·크기·무게 같은 여러 인상과 동시에 연결되기 때문이다[성제현, 2018]. 그래서 결과를 단일 분류가 아니라 Top-3 인상 묶음으로 제시한다.

**인상 축 ↔ 문헌·feature 매핑.** 아래 표는 각 축이 어떤 문헌 근거와 어떤 구현 feature에 연결되는지 보여준다. **문헌 근거**는 "어떤 음운 특징을 볼지"의 출처이고, **구현 feature**의 가중치 숫자는 문헌 값이 아니라 초기 휴리스틱이다(§7).

| 인상 축 | 관련 음운 특징 | 문헌 근거 | 구현 feature |
|---|---|---|---|
| 밝음·가벼움 | 양성 모음 | 성제현(2018), 국립국어원(1993) | bright_vowel_ratio |
| 어두움·무거움 | 음성 모음 | 성제현(2018) | dark_vowel_ratio |
| 작음·섬세함 | 양성 모음 + 낮은 받침 | 성제현(2018) | bright_vowel_ratio, (낮은) final_consonant_density |
| 큼·둔중함 | 음성 모음 + 받침 | 성제현(2018) | dark_vowel_ratio, final_consonant_density |
| 강함·격렬함 | 된소리·거센소리 | 국립국어원(1993) | tense_consonant_ratio, aspirated_consonant_ratio |
| 부드러움·흐름 | 비음·유음 | 국립국어원(1993) | nasal_liquid_ratio |
| 반복·리듬 | 완전/부분 반복 | 국립국어원(1993) | syllable_repetition, char_repetition |
| 지속·여운 | 반복 지속·종성 울림·말줄임 | 국립국어원(1993), 박동근(2014) | ellipsis_energy, final_ng_ratio |
| 변화·교체감 ※ | 자음/모음 교체 | 박동근(2014), 국립국어원(1993) | (미구현) 교체 탐지 |
| 막힘·단절감 | 받침 밀도·어말 자음 | 박동근(2014) | final_consonant_density, final_s_ratio |

## 5. 처리 흐름과 출력 예시

```text
표현 입력
→ 입력 전처리(한글/문장부호/표기 보조 신호 분리)
→ 한글 완성형 음절 분해(초성·중성·종성)
→ 음운 특징 추출 → 14차원 feature vector
→ 해석 카테고리별 점수 계산
→ Top-3 해석 후보 산출
→ 해석 이유와 신뢰도 출력
→ (선택) 해석 결과에 맞는 카모지 등 표시 요소 연결
```

출력 예시:

```text
입력: 아싸아싸!!

해석 후보:
1. 밝음·가벼움
2. 반복·리듬
3. 강함·격렬함

해석 이유:
- 밝은 모음 비율이 높아 밝고 가벼운 인상 점수가 올라갔습니다.
- 같은 음절 패턴이 반복되어 반복·리듬 인상이 감지되었습니다.
- 느낌표가 있어 강도 신호가 보조적으로 반영되었습니다.

신뢰도: high
```

## 6. 구현 범위

**MVP에서 구현한다.**

- tkinter 기반 입력 화면과 해석 결과 출력
- 한글 완성형 음절 분해
- 14차원 음운 특징 벡터 생성(모음·자음 분류, 받침 밀도·종성 비율, 반복, 문장부호 강도)
- NumPy 기반 해석 카테고리 점수 계산과 Top-3 후보·신뢰도 출력
- 해석 이유 출력
- 가중치와 설정값을 JSON으로 분리

**MVP에서 제외한다.**

- 생성형 AI·LLM·외부 API·네트워크 호출
- 대규모 학습 데이터, 딥러닝 모델 학습, 사용자 개인화
- 카모지 추천을 핵심 기능으로 삼는 구조
- 자음/모음 교체 자동 탐지의 완전 구현
- 한국어 발음 변환(G2P)의 완전 구현
- 해석 결과의 심리학적 정확성 보장

## 7. 수치값과 문헌 근거의 구분

문헌은 **"어떤 음운 특징을 볼 것인가"** 를 정하는 근거다. 반면 `category_weights.json`에 들어가는 실제 가중치 숫자·bias·신뢰도 임계값은 논문에서 직접 가져온 값이 아니라 MVP 구현을 위한 **초기 휴리스틱**이다.

따라서 보고서·발표에서는 다음처럼 말한다.

```text
문헌은 반복, 교체, 모음조화, 밝은/어두운 모음 대비, 종성 인상을
분석할 필요가 있음을 보여준다.
하지만 실제 점수 가중치는 문헌 수치가 아니라,
프로그램 구현을 위해 사람이 설정한 초기값이다.
```

## 8. 평가 기준

평가는 카모지를 잘 추천했는지가 아니라, **음운 인상 해석이 설계 의도와 맞는지**를 본다.

- 해석 카테고리가 문헌 기반 음운 인상 축과 연결되어 있는가
- 출력 카테고리가 감정·상황명이 아니라 음운 인상명인가
- 해석 이유가 실제 feature vector 값과 일치하는가
- Top-3 후보로 복수 인상을 보여주는가
- 짧거나 모호한 입력을 낮은 신뢰도로 표시하는가
- 표기 보조 신호(문장부호·자모 반복)가 쓰였을 때 순수 음운 해석과 구분되는가
- 카모지가 없어도 프로그램의 목표와 결과가 설명되는가

## 9. 한계와 방어 논리

음운상징 연구는 소리와 의미 사이의 경향을 보여주지만, 그 관계가 절대적이라고 보지는 않는다. Dingemanse 등(2016)은 ideophone도 음소만으로 의미가 완전히 결정되지 않으며 운율과 관습이 함께 작동한다고 설명한다. Pimentel 등(2019)도 소리와 의미의 통계적 관련은 관찰되지만 효과 크기는 작고 자연언어에서는 자의성이 크게 작동한다고 본다.

따라서 본 프로젝트는 "음운만으로 감정을 맞히는 프로그램"이라고 주장하지 않는다. 올바른 설명은 다음과 같다.

```text
이 프로젝트는 한국어 의성어·의태어 연구에서 다루는 반복, 교체,
모음조화, 자음 인상 등을 계산 가능한 특징으로 바꾸고,
입력 표현이 어떤 소리 인상을 갖는지 설명하는 프로그램입니다.
카모지는 해석 결과를 보여주기 위한 부가 서비스입니다.
```

## 10. 확장 가능성

다음 항목은 MVP 평가 대상이 아니라 이후 발전 방향이다. 기능을 넓히더라도 핵심 목표는 카모지 추천이 아니라 음운 인상 해석이다.

| 구분 | 확장 항목 | MVP에서 제외하는 이유 |
|---|---|---|
| 확장 축 | 자음/모음 교체 자동 탐지 | `변화·교체감`을 산출하려면 별도 탐지 알고리즘이 필요 |
| 입력 분석 | 표기형과 발음형 비교 분석 | 발음 변환 규칙이 들어가면 범위가 커짐 |
| 입력 분석 | 발음 변환(G2P)을 feature 추출에만 적용 | 표기와 실제 발음 차이를 다룰 수 있으나 MVP 핵심은 아님 |
| 평가 자료 | 해석 카테고리별 예시 표현 수집 | 가중치 조정용 자료가 필요하나 과제 범위를 넘길 수 있음 |
| 출력 | 해석 결과에 맞는 카모지·기호 표현 부가 출력 | 시각적 표시일 뿐 해석 모델의 근거가 아님 |
| 출력 | 입력별 feature vector 시각화 | 발표 보조에는 좋으나 핵심 로직은 아님 |
