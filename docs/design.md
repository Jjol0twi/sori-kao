# 설계 문서: 한국어 음운 특징 기반 소리 인상 해석기

이 문서는 구현 명세다. 배경·카테고리 근거·평가 철학은 [`development-goal.md`](development-goal.md)에 있고, 여기서는 그 목표를 어떻게 계산으로 구현하는지를 다룬다.

## 1. 설계 방향

입력은 카모지를 고르기 위한 것이 아니라 **음운 인상을 해석**하기 위한 것이다. 설계의 중심은 세 가지다.

- 한글 음절을 초성·중성·종성으로 분해한다.
- 음운상징 연구가 다루는 반복·교체·모음조화·자음 강도·종성 인상을 계산 가능한 feature로 만든다.
- feature를 해석 카테고리와 비교해 Top-3 해석 후보와 이유를 출력한다.

## 2. 입력과 출력

**입력.** 사용자는 의성어·의태어·감탄사·구호처럼 음운 인상이 잘 드러나는 한국어 짧은 표현을 입력한다. 일반 문장도 받지만 의미 전체를 해석하지 않고 표면 음운 특징만 보수적으로 분석한다.

```text
아싸아싸!! / 우우... / 빠샤 / 두근두근 / 축하
```

**출력.** 카모지보다 해석 결과를 먼저 출력한다.

```text
입력: 아싸아싸!!

해석 후보:
1. 밝음·가벼움
2. 반복·리듬
3. 강함·격렬함

해석 이유:
- 밝은 모음 비율이 높아 밝고 가벼운 인상 점수가 반영되었습니다.
- 같은 음절 패턴이 반복되어 반복·리듬 인상이 감지되었습니다.
- 느낌표가 있어 강도 신호가 보조적으로 반영되었습니다.

신뢰도: high
```

**화면(tkinter).** 복잡한 추천 카드가 아니라 텍스트 중심 결과 영역으로 둔다. 입력창과 결과 영역으로 구성하고, 입력이 확정되면 **해석 이유(2~4개)를 먼저** 보여준 뒤 그 아래에 1위 인상 축의 카모지를 작음/보통/큼 크기별로 여러 개 표시한다. 신뢰도가 낮으면 후보를 줄이고, 동점이면 두 인상 축을 좌우 두 열로 나눠 보여준다. 카모지/기호 표현은 해석의 부가 표시물이라, 없어도 화면이 완결되어야 한다. 입력창 복사/붙여넣기/잘라내기/전체선택은 상단 **편집 메뉴**로 제공한다(macOS에서 uv의 standalone Tk가 Cmd/Ctrl 단축키 이벤트를 전달하지 않아, 키보드 단축키 대신 메뉴를 기본 경로로 둔다 — §12 리스크).

## 3. 전체 처리 흐름

```text
사용자 입력
→ TextPreprocessor
→ HangulDecomposer
→ FeatureExtractor
→ InterpretationScoringModel
→ ExplanationBuilder
→ 해석 결과 출력
```

| 단계 | 역할 |
|---|---|
| TextPreprocessor | 입력 정리, 공백 처리, 한글/문장부호/표기 보조 신호 분리 |
| HangulDecomposer | 한글 완성형 음절을 초성·중성·종성으로 분해 |
| FeatureExtractor | 모음·자음 계열, 받침, 반복, 문장부호 특징 계산 |
| InterpretationScoringModel | NumPy로 feature vector와 해석 카테고리 점수 계산 |
| ExplanationBuilder | 어떤 특징 때문에 그 인상으로 해석되었는지 문장 생성 |

카모지 추천은 핵심 흐름에서 제외한다. 필요하면 해석 결과를 받은 뒤 **별도 표시 단계**에서만 붙이며, 이 단계는 해석 모델의 근거도 평가 대상도 아니다.

## 4. 한글 분해와 입력 처리

완성형 음절(`가`~`힣`)을 유니코드 계산으로 분해한다.

```text
syllable_index = ord(char) - ord('가')
초성 index = syllable_index // (21 * 28)
중성 index = (syllable_index % (21 * 28)) // 28
종성 index = syllable_index % 28
```

비완성형 문자 처리:

| 문자 종류 | 예시 | 처리 |
|---|---|---|
| 완성형 음절 | `아`, `힘` | 초성·중성·종성으로 분해, 분석 음절 수 `N`에 포함 |
| 호환 자모 | `ㅋ`, `ㅠ`, `ㅎ` | 음운 분해 대상 아님. 표기 보조 신호(자모 반복)로만 별도 기록(§8) |
| 문장부호 | `!`, `?`, `...` | 강도·의문·여운 feature로만 사용 |
| 영문·숫자·이모지 | `lol`, `123`, `🔥` | 해석 feature에 사용하지 않음 |

## 5. 음운 특징 벡터

MVP feature vector는 14차원이다.

| feature | 의미 | 범위 |
|---|---|---|
| bright_vowel_ratio | 밝은/양성 모음 비율 | 0~1 |
| dark_vowel_ratio | 어두운/음성 모음 비율 | 0~1 |
| neutral_vowel_ratio | 중성 모음 비율 | 0~1 |
| tense_consonant_ratio | 된소리 초성 비율 | 0~1 |
| aspirated_consonant_ratio | 거센소리 초성 비율 | 0~1 |
| nasal_liquid_ratio | 비음·유음 초성 비율 | 0~1 |
| final_stop_ratio | 불파 폐쇄음 받침(ㄱㄷㅂ류) 비율 | 0~1 |
| final_ng_ratio | ㅇ 받침 비율 | 0~1 |
| final_sonorant_ratio | 공명음 받침(ㄴㅁㄹ) 비율 | 0~1 |
| syllable_repetition | 음절 반복 비율 | 0~1 |
| char_repetition | 문자 반복 비율 | 0~1 |
| exclaim_energy | 느낌표 강도 | 0~1 |
| question_energy | 물음표 강도 | 0~1 |
| ellipsis_energy | 말줄임표 강도 | 0~1 |

### feature 계산식

`N` = 분석 대상 완성형 음절 수. `N = 0`이면 음절 기반 비율 feature는 모두 0으로 둔다(0-나눗셈 방어).

```text
bright_vowel_ratio        = 밝은 모음을 가진 음절 수 / N
dark_vowel_ratio          = 어두운 모음을 가진 음절 수 / N
neutral_vowel_ratio       = 중성 모음을 가진 음절 수 / N
tense_consonant_ratio     = 된소리 초성 음절 수 / N
aspirated_consonant_ratio = 거센소리 초성 음절 수 / N
nasal_liquid_ratio        = 비음·유음 초성 음절 수 / N
final_stop_ratio          = 불파 폐쇄음 받침(음절말 중화로 ㄱ/ㄷ/ㅂ가 되는 받침)을 가진 음절 수 / N
final_ng_ratio            = 종성이 ㅇ인 음절 수 / N
final_sonorant_ratio      = 공명음 받침(ㄴ·ㅁ·ㄹ)을 가진 음절 수 / N
                            (받침을 '있다/없다'가 아니라 불파 폐쇄음 vs 공명음 종류로 나눈다; ㅇ은 final_ng로 분리)
syllable_repetition       = 인접 반복 블록(길이 1~3)에 포함된 음절 수 / N
char_repetition           = 3회 이상 연속 반복된 문자 수 / 전체 문자 수
                            (완성형 음절이 있을 때만 계산, 자모·부호만이면 0)
exclaim_energy            = min(느낌표 개수, 3) / 3
question_energy           = min(물음표 개수, 3) / 3
ellipsis_energy           = min(말줄임표 개수, 3) / 3   (… 또는 마침표 3개 이상)
```

feature **선택**과 가중치의 **부호·상대 크기**는 모두 문헌에 근거한다(§7, §8). 다만 음성상징은 결정 법칙이 아니라 경향이므로(Pimentel 2019; Dingemanse 2015) 절대 수치가 아니라 상대 순위로만 읽고 신뢰도로 완화한다.

## 6. 모음·자음 해석 기준

### 모음 (중성 21개)

| 분류 | 중성 | 해석 |
|---|---|---|
| 밝은/양성 | ㅏ ㅐ ㅑ ㅒ ㅗ ㅘ ㅙ ㅚ ㅛ | 밝음, 가벼움, 작음, 외향적 인상 |
| 어두운/음성 | ㅓ ㅔ ㅕ ㅖ ㅜ ㅝ ㅞ ㅟ ㅠ | 어두움, 무거움, 큼, 낮은 인상 |
| 중성 | ㅣ ㅡ ㅢ | 방향성이 약한 보조 인상 |

### 자음

| 분류 | 자음 | 해석 |
|---|---|---|
| 된소리 | ㄲ ㄸ ㅃ ㅆ ㅉ | 강함, 날카로움, 격렬함 |
| 거센소리 | ㅋ ㅌ ㅍ ㅊ | 에너지, 파열감, 강한 움직임 |
| 비음 | ㄴ ㅁ | 부드러움, 울림 |
| 유음 | ㄹ | 흐름, 미끄러짐, 완만함 |
| 종성 — 불파 폐쇄음 | 받침 ㄱ·ㄷ·ㅂ류 | 울림 없이 끊김 → 막힘·단절감 |
| 종성 — 공명음 | 받침 ㅇ / ㄴ·ㅁ·ㄹ | 울려 이어짐 → 여운·부드러움 |

초성·종성 자음 모두 발음 인상으로만 본다. 초성 `ㅇ`은 소리값이 없어 비음·유음에 넣지 않고, `ㅎ`도 초성에서는 어느 부류에도 강하게 속하지 않아 제외한다. 받침은 '있다/없다'가 아니라 **종류**로 본다: 음절말 중화로 ㄱ·ㄷ·ㅂ가 되는 불파 폐쇄음은 소리가 급정지하고(막힘·단절), 공명음 ㅇ·ㄴ·ㅁ·ㄹ은 울려 이어진다(여운·부드러움). — 박동근(1999) 종성 대립.

## 7. 해석 카테고리와 가중치 설계

카테고리는 감정·대화 상황이 아니라 음운 인상 범주다. 이론 범주는 **10축**, MVP에서 실제 Top-3 산출에 쓰는 것은 **9축**이다. `변화·교체감`(※)은 교체 탐지 feature가 필요해 확장 축으로만 둔다. 각 축의 의미와 문헌 근거는 기획서 §4를 참고한다.

| 카테고리 | 강하게 볼 feature | 약하게 볼 feature | 근거 |
|---|---|---|---|
| 밝음·가벼움 | bright_vowel_ratio | exclaim_energy | 박동근(1999:371), Akita(2021) |
| 어두움·무거움 | dark_vowel_ratio | ellipsis_energy | 박동근(1999:371) |
| 작음·섬세함 | bright_vowel_ratio | 낮은 tense/aspirated ratio | Akita(2021) 한국 음/양 |
| 큼·둔중함 | dark_vowel_ratio | final_ng_ratio, final_sonorant_ratio | 박동근(1999:371) |
| 강함·격렬함 | tense_consonant_ratio, aspirated_consonant_ratio | final_stop_ratio, exclaim_energy | 박동근(1999:374,376) |
| 부드러움·흐름 | nasal_liquid_ratio, final_sonorant_ratio | final_ng_ratio, 낮은 tense/aspirated | 박동근(1999) ㅇ종성, 강석한(2018) |
| 반복·리듬 | syllable_repetition, char_repetition | exclaim_energy | 중첩 도상성(국립국어원 1993) |
| 지속·여운 | ellipsis_energy, final_ng_ratio | final_sonorant_ratio, syllable_repetition | 박동근(1999) ㅇ=여운 |
| 변화·교체감 ※ | (확장) 자음/모음 교체 | — MVP 미산출 | 박동근(2014) |
| 막힘·단절감 | final_stop_ratio | — | 박동근(1999) 폐쇄 종성=단절 |

※ `변화·교체감`은 교체 탐지가 MVP에 없어 점수를 내지 않는다. 설계 카테고리에는 남겨 두어 이후 교체 탐지를 추가하면 활성화한다.

## 8. 점수 계산과 표기 보조 신호

```text
interpretation_scores = W · (x ⊙ informativeness) + bias + auxiliary_bonus
```

- `x`: 14차원 feature vector
- `informativeness`: 자질별 **정보량(markedness) 가중**. 한국어 출현빈도의 역수에 기반해 흔한 모음(특히 ㅏ)은 낮추고 드문 된소리·거센소리·첩어는 키운다. 흔한 ㅏ가 일상어를 밝음으로 쏠리게 하던 '모음 과대'를 **단어 사전·예외 없이** 완화한다. 수치는 빈도 근거 추정(국립국어원 사용 빈도 조사·한국어 사전 표제어 음소 빈도; 코퍼스 정밀값으로 대체 가능, 추정 명시).
- `W`: 해석 카테고리별 feature 가중치
- `bias`: 카테고리 기본값
- `auxiliary_bonus`: **표기 보조 신호**(자모 반복)에만 더한다. 단어 뜻을 보는 의미 힌트가 아니다(아래 매핑 참고).

가중치의 **부호와 상대 크기**는 한국어 음운상징 문헌에 근거한다(자세한 출처는 기획서 §4·§7). 예: 막힘·단절감은 불파 폐쇄음 받침(`final_stop_ratio`)에서만 나오고 공명음 받침은 오히려 여운·부드러움으로 가는데, 이는 박동근(1999)의 종성 대립("ㄱ종성=순간적 단절 vs ㅇ종성=울림·여운")을 그대로 옮긴 것이다. 다만 절대 수치 자체가 논문 값은 아니며, 음성상징의 효과크기가 작다는 반론(Pimentel 2019)에 따라 상대 순위로만 해석하고 신뢰도로 완화한다. 문장부호(느낌표·물음표·말줄임표) 가중치는 음운 문헌 근거가 약한 표기 휴리스틱으로 남긴다.

### 표기 보조 신호 매핑

호환 자모 반복(`ㅋㅋㅋ`, `ㅠㅠㅠ`)은 완성형 음절이 아니어서 음운 feature가 모두 0이다. 이런 입력을 완전히 0점으로 두면 해석이 비므로, 다음처럼 제한적으로 보조한다.

| 표기 패턴 | 보조 인상 축 | 설명 |
|---|---|---|
| `ㅋ`, `ㅎ` 등 자모 2회 이상 반복 | 반복·리듬 | 완성형은 아니지만 반복 표면 신호가 분명함 |
| `ㅠ`, `ㅜ` 등 모음형 자모 2회 이상 반복 | 반복·리듬, 지속·여운 | 길게 이어지는 표기처럼 보여 여운을 약하게 보조 |

특정 자모의 음가(`ㅋ`=거센소리, `ㅠ`=어두운 모음)를 인상 축으로 환산하지 않는다 — "반복되는 표기"라는 표면 신호로만 본다. 즉 `ㅋㅋㅋ`을 감정명으로 해석하지 않고, 음운 정보가 부족한 입력에 `반복·리듬` 중심의 낮은 신뢰도 해석을 줄 뿐이다. 문장부호 강도는 이미 feature vector(exclaim/question/ellipsis_energy)에 들어 있으므로 보너스로 중복 반영하지 않는다.

## 9. 신뢰도와 약신호

다음 중 하나라도 해당하면 `low`로 표시한다.

| 조건 | 의미 |
|---|---|
| 분석 음절 수가 너무 적음 | 해석할 음운 정보 부족 |
| 정보량(유표) 신호가 약함 | 정보량 가중 점수의 top이 임계(weak_signal_threshold=0.7) 미만이고 자모 패턴도 없으면 약신호다. 흔한 모음만으로는 인상을 단정하지 않는다(모음 과대 방지) |
| 표기 보조 신호가 음운 점수보다 큼 | 순수 음운 해석보다 자모 반복 신호에 의존 |
| 모든 점수가 낮거나 0 이하 | 유효한 해석 근거 부족 |

**약신호(`weak_signal`)** 는 위 둘째 조건(유표 신호 부족)을 가리키는 별도 플래그다. 의성어·의태어가 아닌 일반 표현이 주로 여기 해당하며, 화면에는 **"뚜렷한 음운 인상 없음(약한 경향·참고용)"** 으로 결과 맨 위에 명시해 약한 결과를 단정으로 오해하지 않게 한다. 손으로 고른 강신호 목록 대신 **정보량 가중 점수의 크기**로 균일 판정한다(§8) — 특정 단어용 예외 필터가 아니다.

1위와 2위 점수 차이가 작다는 이유만으로 `low`를 주지는 않는다. 이 프로젝트의 인상 축은 상호배타적 분류가 아니므로, 여러 축이 비슷하게 높게 나오는 경우는 모호함이 아니라 **복합 인상**으로 본다.

`low`는 실패를 숨기는 장치가 아니라 과도한 확신을 피하기 위한 표시다.

## 10. 해석 이유 생성

해석 이유는 실제 점수 기여도에 기반한다.

```text
contribution_i = x_i * W[category][i]
```

1위 카테고리에서 기여도가 큰 feature 2~4개를 설명에 쓴다. 표기 보조 신호가 쓰였으면 "표기 보조 신호로 반영되었습니다"라고 표시해 순수 음운 해석과 구분한다.

| feature | 설명 문장 |
|---|---|
| bright_vowel_ratio 높음 | 밝은 모음 비율이 높아 밝음·가벼움 인상 점수가 반영되었습니다. |
| dark_vowel_ratio 높음 | 어두운 모음 비율이 높아 어두움·무거움 인상 점수가 반영되었습니다. |
| tense_consonant_ratio 높음 | 된소리 비율이 높아 강함·격렬함 인상 점수가 반영되었습니다. |
| nasal_liquid_ratio 높음 | 비음·유음 비율이 높아 부드러움·흐름 인상 점수가 반영되었습니다. |
| syllable_repetition 높음 | 같은 음절 패턴이 반복되어 반복·리듬 인상이 감지되었습니다. |
| ellipsis_energy 높음 | 말줄임표가 감지되어 지속·여운 인상이 보조적으로 반영되었습니다. |

## 11. 평가 계획

평가는 카모지가 아니라 해석 결과를 기준으로 한다.

| 테스트 입력 | 기대 해석 | 확인할 점 |
|---|---|---|
| `아싸아싸!!` | 밝음·가벼움, 반복·리듬, 강함·격렬함 | 밝은 모음과 반복이 반영되는가 |
| `우우...` | 어두움·무거움, 지속·여운 | 어두운 모음과 말줄임표가 반영되는가 |
| `빠샤!` | 강함·격렬함, 밝음·가벼움 | 거센소리와 느낌표가 반영되는가 |
| `똑딱똑딱` | 강함·격렬함, 반복·리듬, 막힘·단절감 | 불파 폐쇄음 받침(ㄱ)이 막힘으로, 된소리가 강함으로 반영되는가 |
| `안녕` | 부드러움·흐름, 지속·여운 + 낮은 신뢰도 | 공명 받침(ㄴ·ㅇ)이 막힘이 아니라 여운·부드러움으로 가는가 |
| `하암` | 밝음·가벼움 + 낮은 신뢰도 | 음운만으로 하품 의미를 단정하지 않는가 |
| `축하` / `추카` | 표기/발음 차이 | 폐쇄 받침(ㄱ)·거센소리 차이가 해석을 바꾸는가 |

`축하`/`추카`는 자동 G2P 변환 테스트가 아니라, 두 표기를 각각 입력해 feature 차이를 비교하는 **수동 시연**이다. 발음 변환을 확장 기능으로 둘 가치가 있음을 보여주는 예시다.

## 12. 리스크

| 리스크 | 설명 | 대응 |
|---|---|---|
| 감정 분류로 오해 | 밝음·무거움 같은 인상을 감정명으로 단정할 위험 | 감정명이 아니라 음운 인상명으로 출력 |

| 가중치 근거 공격 | 절대 수치까지 논문 값은 아님 | 부호·상대 크기는 문헌(박동근 1999 등) 근거, 효과크기가 작다는 반론(Pimentel 2019)은 상대 순위+신뢰도로 수용 |

## 13. 완료 기준

- 입력 표현을 한글 음절 단위로 분해한다.
- 최소 10개 이상의 음운 특징을 숫자 벡터로 만든다.
- 논문 기반 음운 인상 카테고리의 Top-3를 출력한다.
- 짧거나 모호한 입력, 표기 보조 신호 의존 입력은 낮은 신뢰도로 표시한다.
- 해석 이유가 실제 feature contribution과 일치한다.
- 카모지 추천 없이도 프로그램 목표가 설명된다.
- 음운상징을 감정 정답으로 과장하지 않는다.
- 생성형 AI/API/외부 네트워크 없이 실행된다.
